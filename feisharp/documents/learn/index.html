FeiSharp 编译器错误和警告

FeiSharp 版本 8.0

此文档列出了 FeiSharp 编译器生成的所有错误和警告。错误代码以 "FS" 开头，后跟四位数字。第一位数字表示错误类别：

  • 1xxx: 词法分析错误（Lexer 错误）
  • 2xxx: 语法分析错误（Parser 错误）
  • 3xxx: 语义错误
  • 4xxx: 运行时错误
  • 5xxx: 编译器警告

词法分析错误 (1xxx)

这些错误在词法分析阶段发生，当 Lexer 无法将源代码字符转换为有效的词法单元时触发。

错误

FS1001: 无效字符

FS1001

Lexer 遇到了无法识别的字符。

原因: 源代码中包含 Lexer 无法处理的字符（如特殊符号、控制字符或不受支持的Unicode字符）。

示例代码
var x = 10;
var y = 20 @ 30;  // 错误：@ 是无效字符
print(x + y);
控制台输出
错误 FS1001: 第2行，位置12: 无效字符 '@'
Unexpected character: @, it will be skipped
解决方法

删除或替换无效字符。如果确实需要使用特殊字符，请确保它被包含在字符串字面量中：

var message = "符号 @ 可以在字符串中使用";
print(message);
错误

FS1002: 未终止的字符串字面量

FS1002

字符串字面量缺少结束引号。

原因: 双引号字符串没有以匹配的双引号结束。

示例代码
var message = "这是一个未终止的字符串;
print(message);
控制台输出
错误 FS1002: 第1行: 未终止的字符串字面量
注意

Lexer 会一直读取字符直到遇到结束引号或文件结束。这可能导致后续代码被错误地解析为字符串的一部分。

错误

FS1003: 无效的数字格式

FS1003

数字字面量格式不正确。

原因: 数字包含多个小数点或包含非数字字符。

示例代码
var a = 3.14.15;    // 错误：多个小数点
var b = 123abc;     // 错误：数字中包含字母
var c = 1.2.3;      // 错误：无效的数字格式
控制台输出
错误 FS1003: 第1行，位置10: 无效的数字格式 '3.14.15'
错误 FS1003: 第2行，位置10: 无效的数字格式 '123abc'
警告

FS1004: 未知字符警告

FS1004

Lexer 遇到了不受支持的字符，但选择跳过它继续解析。

原因: 这是 Lexer 的错误处理机制，当遇到无法识别的字符时会输出警告并跳过该字符。

示例代码
var x = 10;
# 这是一个注释，但 # 不是合法的注释符号
print(x);
控制台输出
警告 FS1004: 第2行，位置1: 未知字符 '#'，已跳过
Unexpected character: #, it will be skipped
Lexer 源代码实现

在 Lexer 的 NextToken() 方法中，当所有已知字符模式都不匹配时：

// Lexer.cs - 错误处理部分
Console.WriteLine("Unexpected character: " + current + ", it will be skipped");
_index++;

语法分析错误 (2xxx)

这些错误在语法分析阶段发生，当 Parser 无法根据语法规则构建语法树时触发。

错误

FS2001: 缺少分号

FS2001

语句缺少结束分号。

原因: FeiSharp 要求大多数语句以分号结束。

示例代码
var x = 10  // 错误：缺少分号
var y = 20;
print(x + y)
控制台输出
错误 FS2001: 第1行: 语句缺少分号
错误 FS2001: 第3行: 语句缺少分号
语法规则

需要分号的语句包括：变量声明、赋值语句、函数调用、表达式语句等。

错误

FS2002: 括号不匹配

FS2002

圆括号、方括号或花括号不匹配。

原因: 左括号没有对应的右括号，或者括号类型不匹配。

示例代码
var result = (10 + 5 * 2;  // 错误：缺少右括号
if x > 5 {               // 错误：条件缺少括号
    print("大于5");
]                       // 错误：错误的括号类型
控制台输出
错误 FS2002: 第1行: 缺少 ')'
错误 FS2002: 第2行: 条件表达式缺少 '('
错误 FS2002: 第4行: 意外的 ']'，期望 '}'
Parser 实现参考

Parser 使用栈来跟踪括号匹配：

// Parser.cs - 括号匹配逻辑
private void ExpectPunctuation(string expected)
{
    var token = GetCurrentToken();
    if (token.Value != expected)
    {
        throw new ParseException($"期望 '{expected}'，但找到 '{token.Value}'");
    }
    Advance();
}
错误

FS2003: 无效的语法

FS2003

代码不符合 FeiSharp 语法规则。

原因: 语句结构不正确，或者使用了不支持的语法结构。

示例代码
var = 10;              // 错误：缺少变量名
function add a, b)    // 错误：缺少左括号
    return a + b;
fend;

if x > 5 then         // 错误：无效的 if 语法
    print(x);
iend;
控制台输出
错误 FS2003: 第1行: 无效的变量声明语法
错误 FS2003: 第2行: 期望 '('
错误 FS2003: 第6行: 无效的条件语句语法
错误

FS2004: 无效的块结构

FS2004

代码块结构不正确。

原因: FeiSharp 使用特定的关键字标记代码块开始和结束（如 fbegin/fend, ibegin/iend, wbegin/wend）。

示例代码
function add(a, b)
begin:           // 错误：应该是 fbegin
    return a + b;
end;            // 错误：应该是 fend

if (x > 5)
ibegin:
    print(x);
wend;           // 错误：应该是 iend
控制台输出
错误 FS2004: 第2行: 期望 'fbegin'
错误 FS2004: 第4行: 期望 'fend'
错误 FS2004: 第9行: 期望 'iend'，但找到 'wend'

语义错误 (3xxx)

这些错误在语义分析阶段发生，代码语法正确但含义有问题。

错误

FS3001: 未定义的变量

FS3001

使用了未声明的变量。

原因: 变量在使用前没有用 var 关键字声明。

示例代码
x = 10;             // 错误：x 未定义
print(y);          // 错误：y 未定义

var z = a + b;     // 错误：a 和 b 未定义
控制台输出
错误 FS3001: 第1行: 未定义的变量 'x'
错误 FS3001: 第2行: 未定义的变量 'y'
错误 FS3001: 第4行: 未定义的变量 'a'
错误 FS3001: 第4行: 未定义的变量 'b'
正确的变量使用

所有变量必须先声明后使用：

var x = 10;        // 正确：声明并初始化
var y;             // 正确：声明（需要后续初始化）
init(y, Number);   // 初始化
print(x + y);      // 正确：使用已声明的变量
错误

FS3002: 类型不匹配

FS3002

操作或赋值中的类型不兼容。

原因: FeiSharp 是动态类型语言，但某些操作要求类型兼容。

示例代码
var x = "字符串";
var y = 10;
var z = x + y;     // 可能产生警告：字符串和数字相加

var a = true;
var b = a + 1;     // 错误：布尔值和数字不能相加
控制台输出
警告 FS5001: 第3行: 字符串和数字相加，结果将被转换为字符串
错误 FS3002: 第6行: 类型不匹配，不能将数字加到布尔值
错误

FS3003: 函数参数数量不匹配

FS3003

函数调用提供的参数数量与函数定义不匹配。

原因: 调用函数时传递了错误数量的参数。

示例代码
function add(a, b)
fbegin:
    return a + b;
fend;

var result1 = add(5);          // 错误：缺少一个参数
var result2 = add(1, 2, 3);    // 错误：多了一个参数
控制台输出
错误 FS3003: 第7行: 函数 'add' 期望 2 个参数，但提供了 1 个
错误 FS3003: 第8行: 函数 'add' 期望 2 个参数，但提供了 3 个

运行时错误 (4xxx)

这些错误在代码执行期间发生。

错误

FS4001: 除零错误

FS4001

试图除以零。

原因: 除法运算的除数为零。

示例代码
var x = 10;
var y = 0;
var z = x / y;     // 运行时错误：除零
控制台输出
运行时错误 FS4001: 除零错误
   at line 3: var z = x / y;
预防措施

在执行除法前检查除数：

var x = 10;
var y = 0;
if (y != 0)
ibegin:
    var z = x / y;
iend:
else
ibegin:
    print("除数不能为零");
iend;
错误

FS4002: 空引用错误

FS4002

试图访问 null 值的成员或调用 null 值的函数。

示例代码
var obj = null;
var value = obj.property;     // 运行时错误：空引用
错误

FS4003: 索引越界

FS4003

数组或字符串索引超出有效范围。

示例代码
var arr = [1, 2, 3];
var value = arr[5];           // 运行时错误：索引越界

编译器警告 (5xxx)

这些是编译器警告，代码可以编译和执行，但可能存在潜在问题。

警告

FS5001: 隐式类型转换

FS5001

编译器检测到可能不安全的隐式类型转换。

示例代码
var x = "100";
var y = x + 50;     // 警告：字符串和数字相加
警告

FS5002: 未使用的变量

FS5002

声明了变量但从未使用。

示例代码
var unused = 10;    // 警告：变量未使用
var x = 5;
print(x);
警告

FS5003: 缺少返回值

FS5003

函数可能在某些路径下没有返回值。

示例代码
function check(x)
fbegin:
    if (x > 0)
    ibegin:
        return true;
    iend:
    // 警告：如果 x <= 0，函数没有返回值
fend;
